## What shipped today

### Lessons / learning
- **Voice conversation mode** — deployed (was built but unpushed). STT (browser Web Speech API) + OpenAI TTS, auto-advance loop, in `app/lesson/page.jsx`.
- **Narrated video lessons** — new "Watch (narrated)" mode on `/lesson`. `lib/ai.js` `generateVideoScript`, `app/api/lesson/video/route.js`, `components/video-lesson-player.jsx`. Uses OpenAI TTS.
- **Skill Graph surfaced** — "View Skill Graph" link added to the homepage "Your Skills" card (`app/page.jsx`); the graph already existed at `/skill-graph`.

### Manager dashboard (Snowflake via n8n)
- **Flipped from push → webhook.** The old n8n "push to /api/manager-data with x-api-secret" 401'd. Now Vercel **pulls** from an n8n webhook (same direction as the working `manager-lookup`), caches to Blob for 6h.
  - n8n workflow file: `n8n-manager-dashboard-webhook.json` (webhook `POST /manager-dashboard` → Snowflake → aggregate → respond). **Active in n8n.**
  - Vercel: `lib/manager-data.js` (`getOrgData`), `app/api/manager-data/route.js` (GET pulls + caches; `?refresh=1` forces).
  - Env: `N8N_MANAGER_DASHBOARD_WEBHOOK_URL` (set in Vercel prod).
  - **Snowflake gotcha:** unquoted aliases return UPPERCASE columns; aggregate code reads `emp.FULL_NAME` etc. (see `reference_snowflake_n8n_uppercase_columns` memory).
- **Manager lookup served from cache** — the old separate `manager-lookup` webhook is retired; `/api/manager-lookup` now resolves a manager + reports from the cached org data (single workflow). Only people *with* reports resolve (by design).

### XP system
- **Fixed XP never accruing in demo mode** — `profile.id` is empty in demo, so `if (profile?.id)` guards skipped all writes/reads. New `lib/learner-id.js` `resolveLearnerId(profile)` used in `progression-provider`, `lesson`, `chat`.
- **Unified XP** — quests & review were writing to a separate `xp-store` the Level bar never read; routed both through the progression system; **deleted `lib/xp-store.js`**. All four sources (lessons, chat, quests, review) now feed one store.
- ⚠️ Persistence is still **localStorage per browser** — `saveToBlob` writes but nothing reads it back. Cross-device sync needs Supabase.

### Find AI vs Just Chat (separated)
- The "Find AI for your work" hero routed to `/chat`, conflating the two. Now routes to **`/discover`** (the opportunity finder), passing the work description as `?q=`.
- `/discover` **prefills but does NOT auto-run** — user reviews/edits, then hits "Find AI for me". Hint nudges them to update saved tasks.
- **Just Chat** (`/chat`) reverted to a plain LLM-chat framing ("What do you want to know about AI?").
- **Round-trip:** Discover "Update your tasks" → `/my-tasks?from=discover` → "Save & continue" returns to `/discover` with refreshed tasks; "Back to Find AI for your work" link to leave without saving.
