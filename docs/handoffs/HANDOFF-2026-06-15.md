## What shipped today

### 1. Multi-goal selection (unlimited)
- Onboarding "What's your main goal?" → **"What are your goals?"** — checkbox multi-select, no cap (`app/onboarding/page.jsx`, `StepGoal`).
- **My Role** goal picker is multi-select too; display shows "Goals: A, B, C" (`app/my-role/page.jsx`).
- **Data model:** profile now stores a `goals` array **plus** a joined `goal` string (`goals.join('; ')`) kept in sync. This means every existing lesson/AI-prompt read site (`lib/ai.js` ×5, `lib/track.js`, `app/api/quick-win`) keeps working untouched — they read `profile.goal` and now get all goals. `extractRole` (`lib/role-manager.js`) derives `goals` with a fallback: `profile.goals` → `[profile.goal]` → `[]`, so old single-goal profiles don't break. `lib/data.js` also passes `goals` through.

### 2. Tour walks through every menu section
- Added `data-tour` anchors to each sidebar section header (`section-learn`, `section-progress`, `section-manager`, `section-settings`, `section-skillshop`, `section-admin`) and to the header (`home-link`, `name-menu`).
- `lib/menu-walkthrough-steps.js` rewritten to step through Home → Learn → Your Progress → Manager → Settings → HCP Skill Shop → your name → Help. Admin intentionally excluded (only exists for admins, would float as a stray modal otherwise).

### 3 + 4. Menu cleanup, Settings section, name dropdown
- **Account section dissolved** (`components/sidebar.jsx`).
- New **Settings** section: Dark mode, Tour, Voice.
- **Manager** kept in the menu as a one-item section labeled "Team Dashboard".
- **Name dropdown** (`components/user-menu.jsx`) — top-right on *every* page now (added `UserMenu` to `components/page-header.jsx`; dashboard already had it). Holds Profile / My Role / My Tasks / Projects + Switch user / Log out. Switch-user + log-out logic moved here from the sidebar (sidebar's bottom session block + its `softLogin`/`handleSwitchUser`/`signOut` removed).

### 5. Dashboard → Home
- Renamed and promoted to a **persistent header link** (`Home` icon + label in `UserMenu`, so it renders via `PageHeader` and the dashboard header). Removed from the hamburger. Reachable without opening the menu.

### 6. Help widget keeps your text
- `components/help-widget.jsx` now persists `{ messages, input, open }` to `sessionStorage` (`la_help_widget`), hydrating on mount. Typed text + conversation survive navigation between pages. (Previously lost on most pages.)

### 7. Slack channels in Just Chat
- `app/chat/page.jsx` footer now pins the two support channels (HCP Skill Shop Help `C04BU29V4TH`, HCP MX Skill Shop Help `C04J8BRUJQY`), matching the help widget.

### 8. Prominent lesson finish + Quick Tip XP clarity
- `app/lesson/page.jsx` engagement area: the quiet "Finish lesson" link is now a primary button **"Finish {Quick Tip / Quick Lesson / Deep Dive} · +50 XP"** (Trophy icon), inside a highlighted card with copy: *"Keep the conversation going above to learn more — or finish to earn +50 XP."* "Exit without finishing" is the quiet secondary. (Finishing routes through `finishLesson()` → complete phase → `onLessonComplete` → +50 XP.)
